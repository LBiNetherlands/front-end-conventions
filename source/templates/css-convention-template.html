Introduction

In all projects, but especially on large, long-running projects, with multiple developers of differing specialties and abilities, it is important that we work in a unified way in order to:

  • Make sure code and stylesheets are readable for other developers;
  • Give developers a feeling of familiarity across codebases;
  • Keep stylesheets scalable and maintainable;

This guide is meant as a reference for our developers and freelance developers that work for us. All developers working for DigitasLBi should strive in staying loyal to these guidelines. Any deviation from the rules in this styleguide must be fully justified.
